# Parcel Delivery System — Pustovoit 372

## 1. Назва роботи
Проєктування програмних систем: сучасні архітектурні підходи. Варіант 14 - Система оформлення доставки посилки.

## 2. Опис use case
Система дозволяє користувачеві створити нове відправлення для доставки. Під час створення система перевіряє фізичні параметри посилки на відповідність правилам компанії. Якщо параметри є допустимими, система автоматично розраховує кінцеву вартість доставки на основі ваги та зберігає замовлення у базі даних, повертаючи клієнту унікальний трек-номер.

## 3. Обрана архітектура
Для реалізації цього проєкту було обрано гексагональну архітектуру, яка також відома як архітектура портів та адаптерів. Цей підхід дозволяє максимально ізолювати бізнес-правила від технічних деталей фреймворків чи баз даних.

## 4. Обґрунтування вибору
Такий архітектурний стиль ідеально підходить для логістичних систем. Правила розрахунку вартості та обмеження габаритів посилок можуть часто змінюватися бізнесом, тому вони повинні бути захищені від змін у зовнішніх технологіях. Гексагональна архітектура дозволяє нам тестувати ядро системи взагалі без підключення до реальної бази даних або інтерфейсу користувача. Ми можемо легко замінити консольний ввід на веб-інтерфейс, не змінюючи жодного рядка коду в самому сервісі доставки.

## 5. Структура проєкту
Проєкт розділено на три основні пакети, які відображають шари архітектури. Пакет domain містить сутності та бізнес-правила перевірки посилки. Пакет application містить порти та сервіс сценарію використання. Пакет infrastructure містить технічні реалізації взаємодії із зовнішнім світом.

## 6. Розподіл відповідальностей
Ядро системи знаходиться у пакеті domain. Тут класи Parcel та Shipment відповідають за перевірку допустимої ваги, габаритів та збереження стану. Вони нічого не знають про інші шари.

Сценарій використання знаходиться у пакеті application. Клас CreateShipmentService бере на себе роль координатора. Він приймає запит, створює сутності, виконує математичний розрахунок ціни та передає готове відправлення на збереження.

Для зв'язку зі світом використовуються порти. CreateShipmentUseCase є вхідним портом для отримання команд від користувача. ShipmentRepository є вихідним портом для збереження даних.

Адаптери знаходяться у пакеті infrastructure. Клас InMemoryShipmentRepository реалізує вихідний порт і виконує роль сховища даних у оперативній пам'яті. Клас ConsoleUI реалізує вхідний інтерфейс користувача та передає команди у систему.

## 7. Інструкція запуску
Для запуску програми необхідно відкрити проєкт у середовищі розробки. Далі потрібно знайти клас ConsoleUI у пакеті infrastructure. Запустіть метод main у цьому класі. Після запуску у консолі з'являться підказки для введення параметрів посилки та адрес доставки.
